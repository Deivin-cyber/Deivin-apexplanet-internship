OSI Model Layers & Functions

The OSI (Open Systems Interconnection) model explains how data moves from one computer to another in a network. It divides communication into 7 layers, and each layer has a specific role.

Layer 1 – Physical:
This layer handles the physical transfer of data through cables, fiber optics, or wireless signals.

Layer 2 – Data Link:
This layer manages data frames and uses MAC addresses to send data within the same network.

Layer 3 – Network:
This layer is responsible for routing data between networks using IP addresses and sending packets through the best path.

Layer 4 – Transport:
This layer ensures data is delivered correctly and in order. It uses protocols like TCP and UDP.

Layer 5 – Session:
This layer manages the connection between applications, such as starting and ending communication sessions.

Layer 6 – Presentation:
This layer translates and formats data so that the receiving system can understand it. It also handles encryption and compression.

Layer 7 – Application:
This is the layer closest to the user. It provides services for applications like web browsers, email, and file transfers.

TCP/IP Protocol Suite

The TCP/IP model is the practical networking model used on the internet. It defines how data is sent, received, and addressed between devices.

TCP (Transmission Control Protocol):
TCP is a reliable protocol that ensures data is delivered correctly and in the right order. It divides data into packets and reassembles them at the destination.

IP (Internet Protocol):
IP provides a unique IP address to each device on the network and helps route data packets from the sender to the receiver.

DNS & HTTP / HTTPS

DNS (Domain Name System):
DNS works like the phonebook of the internet. It converts domain names (like google.com) into IP addresses so computers can locate websites.

HTTP (Hypertext Transfer Protocol):
HTTP is the protocol used by web browsers to request and receive web pages from servers.

HTTPS (HTTP Secure):
HTTPS is the secure version of HTTP. It uses SSL/TLS encryption to protect data sent between the browser and the website.

Example:

HTTP → http://

HTTPS → https://

IP Addressing, Subnetting, and NAT

IP Addressing:
Every device connected to a network has a unique IP address that helps identify it and deliver data correctly.

Subnetting:
Subnetting divides a large network into smaller networks called subnets. This improves network performance, management, and security.

NAT (Network Address Translation):
NAT allows multiple devices in a private network to use one public IP address to access the internet. This helps save public IP addresses and adds extra security.
